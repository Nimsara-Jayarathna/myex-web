# Blipzo Web v1.0.0 Release Notes

## Overview
- **Version:** v1.0.0
- **Release Date:** 2026-01-01
- **Summary:** Initial production release of the Blipzo Web frontend. This release delivers a modern, responsive personal finance dashboard built with React 19 and Tailwind CSS v4, fully integrated with the Blipzo v1 API for secure expense and income tracking.

## Features
- **Authentication & Security**
  - Secure login and registration via modal interfaces.
  - HttpOnly cookie-based session management with automatic token rotation.
  - Protected route guards ensuring authorized access to dashboard features.

- **Dashboard & Analytics**
  - Interactive dashboard providing a high-level overview of finances.
  - Visual analytics using Chart.js for income vs. expense trends.
  - Summary cards for quick status updates (Total Balance, Income, Expenses).

- **Transaction Management**
  - Full CRUD capabilities for financial transactions.
  - Filterable and searchable transaction lists.
  - Intuitive "Add Transaction" modal for quick entry.

- **Category Management**
  - Custom category creation and management for better financial organization.
  - Color-coded categories for easy visual distinction in charts.

- **Settings & Preferences**
  - Tabbed settings interface for managing application preferences.
  - User profile and account management sections.

- **Modern UI/UX**
  - Fully responsive design suitable for mobile and desktop.
  - Modern aesthetic using Tailwind CSS v4 tokens and utilities.
  - Smooth transitions and interactive elements powered by Framer Motion.

## Bug Fixes
- Addressed initial layout shifts on mobile devices.
- Resolved z-index rendering issues in dropdown menus.
- Fixed toggle pill alignment in settings forms.

## Improvements / Refactors
- **API Versioning**: complete migration to use `/api/v1/` endpoints for all backend communications, ensuring long-term compatibility.
- **Styling Architecture**: Migrated to Tailwind CSS v4 with Vite integration, replacing legacy CDN approaches for better performance and build optimization.
- **State Management**: Implemented efficient global state using Zustand, specifically for authentication handling.
- **Data Fetching**: Integrated React Query for robust server state management, caching, and background updates.

## Known Limitations
- **Backend Dependency**: Requires a running instance of the Blipzo Node.js backend (v1.0.0+) to function.
- **Connectivity**: The application requires an active internet connection; offline mode is currently not supported.

## Setup / Run Instructions (high-level)
1. **Prerequisites**: Node.js v20+, npm.
2. **Installation**:
   ```bash
   npm install
   ```
3. **Configuration**:
   - Create a `.env` file in the root.
   - Set `VITE_API_BASE_URL` to your backend URL (e.g., `http://localhost:3000`).
4. **Development**:
   ```bash
   npm run dev
   ```
5. **Production Build**:
   ```bash
   npm run build
   npm run preview
   ```

## Branching & Versioning Strategy
- **main**: Production-ready code. Deploys to production environment.
- **dev**: Staging/Integration branch for testing new features.
- **feature/*** : Short-lived branches for specific feature implementations.
- **Semantic Versioning**: Adheres to SemVer (MAJOR.MINOR.PATCH).

## Notes
- Ensure the backend CORS configuration allows credentials from this frontend's origin.
- This release is optimized for React 19; ensure development environments interpret peer dependencies correctly.
